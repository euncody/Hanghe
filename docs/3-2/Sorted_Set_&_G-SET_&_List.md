# 데이터 구조: Sorted Set, G-Set, List

## Sorted Set
- 값(value)과 점수(score)를 함께 저장하고 자동 정렬된 집합.
- 대표 구현: Redis ZSET.
- 활용: 랭킹 시스템, 우선순위 큐.

## G-Set (Grow-only Set)
- CRDT의 일종으로 **추가만 가능한 집합**.
- 삭제 불가능 → 충돌 없음.
- 활용: 이벤트 로그, 발생 사실 기록.

## G-Set vs List
| 특징 | **G-Set** | **List** |
|------|-----------|----------|
| 구조 | 집합(Set) → 중복 없음 | 순서 있는 리스트(List) |
| 연산 | 추가만 가능 (삭제 불가) | 삽입, 삭제, 순서 유지 |
| 중복 | 허용하지 않음 | 허용 가능 |
| 순서 | 없음 | 있음 |
| 활용 | 고유 이벤트 기록 | 채팅 메시지, 작업 큐 |
